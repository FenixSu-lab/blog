## 页面一：仪表盘（Dashboard）

### Prompt

```
设计一个企业管理后台的仪表盘页面（Dashboard），用于监控出口信用保险的投保和缴费状态。

页面布局：
- 顶部：页面标题"保险管理仪表盘"，右侧显示当前日期
- 主体区域：上方是 4 个核心指标卡片，下方是一个待办事项表格

核心指标卡片（横向排列，每个卡片包含图标、数值、标签、同比变化）：
1. 本月投保率：显示百分比如 87.5%，低于 90% 显示红色警告图标，高于 90% 显示绿色成功图标
2. 未投保订单数：显示数量如 12 单，有未投保订单时显示红色，全部投保时显示绿色"全部已投保"
3. 待缴费保单：显示数量和金额如"3 单 / ¥45,200"，7 天内到期的保单
4. 逾期未缴费：显示数量如 2 单，有逾期时显示红色并带警告图标，无逾期时显示绿色"无逾期"

下方待办事项表格：
- 表格列：保单号、客户名称、保费金额、缴费截止日、状态（未缴费/已缴费/逾期）、剩余天数、操作
- 状态列用不同颜色标签区分：未缴费（蓝色）、已缴费（绿色）、逾期（红色）
- 剩余天数：正数显示"还剩 X 天"（黄色），负数显示"逾期 X 天"（红色），已缴费显示"—"
- 操作列：未缴费的显示"提醒"按钮，逾期的显示"紧急通知"按钮（红色）
- 表格支持按状态筛选，顶部有筛选按钮组：全部 / 未缴费 / 已缴费 / 逾期

技术要求：使用 React + Tailwind CSS + shadcn/ui 组件库，深色主题，响应式布局。
```

## 页面二：订单投保管理

### Prompt

```
设计一个企业管理后台的订单投保管理页面，用于管理出口贸易和内贸订单的保险投保流程。

页面布局：
- 顶部：页面标题"订单投保管理"，右侧有"一键投保"和"批量投保"两个主操作按钮
- 筛选区域：贸易类型下拉框（全部/出口险/内贸险）、投保状态下拉框（全部/未投保/已申请限额/已限额通过/已投保）、客户名称搜索框、查询和重置按钮
- 主体：数据表格
- 底部分页器

数据表格列：
1. 订单编号：如 SO-2026-00123，可点击跳转详情
2. 客户名称：买方公司名称
3. 订单金额：显示金额和币种，如"USD 50,000.00"
4. 贸易类型：标签显示"出口险"（蓝色）或"内贸险"（紫色）
5. 投保状态：用步骤标签显示
   - 未投保（灰色）
   - 已申请限额（蓝色）
   - 已限额通过（绿色）
   - 已投保（绿色带勾）
6. 保险到期日：如 2026-12-31，已过期显示红色
7. 操作列：
   - 未投保状态：显示"申请限额"按钮（主按钮）
   - 已限额通过：显示"出运申报"按钮（主按钮）
   - 所有状态：显示"详情"链接

交互逻辑：
- 点击"一键投保"按钮弹出确认对话框，显示"将为选中的未投保订单自动申请限额，是否继续？"
- 表格行前面有复选框，支持多选后批量操作
- 点击订单编号跳转到投保详情页

投保详情页（同一个页面，点击订单编号后切换到详情视图）：
- 顶部面包屑：订单投保管理 > 订单详情
- 订单基本信息卡片：订单编号、客户名称、订单金额、贸易类型
- 投保流程进度条：创建订单 → 买方资信调查 → 限额申请 → 限额审批 → 出运申报 → 收汇确认 → 完成，当前步骤高亮显示
- 下方显示每个步骤的详细信息：操作时间、操作人、中信保返回的审批结果、备注
- 右上角有"返回列表"按钮

技术要求：使用 React + Tailwind CSS + shadcn/ui 组件库，深色主题，响应式布局。
```

## 页面三：限额管理

### Prompt

```
设计一个企业管理后台的限额管理页面，用于管理中信保出口信用保险的买方信用限额申请和查询。

页面布局：
- 顶部：页面标题"限额管理"，右侧有"新建限额申请"主按钮
- 筛选区域：买方名称搜索框、国家/地区下拉框、限额状态下拉框（全部/待审批/已通过/已拒绝）、查询和重置按钮
- 主体：数据表格
- 底部分页器

数据表格列：
1. 买方名称：如 ABC Trading Co.
2. 国家/地区：如 美国、德国、日本
3. 限额币种：USD、EUR 等
4. 限额金额：如 USD 500,000.00
5. 贸易类型：非LC无担保 / 非LC企业担保 / 普通信用证 / 保兑信用证
6. 申请日期：如 2026-06-01
7. 审批状态：待审批（黄色标签）、已通过（绿色标签）、已拒绝（红色标签）
8. 操作列：查看详情、编辑（仅待审批状态可编辑）

新建限额申请弹窗（Modal）：
- 弹窗标题：新建限额申请
- 表单字段分两栏布局：
  左栏-买方信息：
  - 买方名称（输入框，必填）
  - 买方国家/地区（下拉框，必填）
  - 买方地址（输入框）
  右栏-交易信息：
  - 限额币种（下拉框：USD/EUR/GBP/JPY，必填）
  - 限额金额（数字输入框，必填）
  - 付款方式（下拉框：D/P、D/A、O/A、L/C，必填）
  - 担保类型（下拉框：无担保、企业担保、银行保函，当付款方式为 O/A 时必填）
  底部-附件上传：
  - 贸易合同上传区域，支持拖拽上传，文件格式：PDF/DOC/JPG，最大 10MB
- 弹窗底部：取消按钮（灰色）、提交申请按钮（蓝色主按钮）
- 表单验证：必填字段未填时显示红色错误提示

限额详情页（点击表格行后展开或跳转）：
- 买方信息卡片：名称、国家、地址
- 限额信息卡片：限额金额、币种、贸易类型、申请日期、审批状态
- 审批记录时间线：显示每个审批节点的时间、操作人、结果、备注
- 右上角：返回列表按钮

技术要求：使用 React + Tailwind CSS + shadcn/ui 组件库，深色主题，响应式布局。
```

## 页面四：缴费监控

### Prompt

```
设计一个企业管理后台的缴费监控页面，用于监控中信保保单的缴费状态和逾期预警。

页面布局：
- 顶部：页面标题"缴费监控"，右侧有"导出报表"按钮
- 统计卡片区域（横向排列 4 个卡片）：
  1. 总保单数：如 156 单，图标为保单文档
  2. 已缴费：如 142 单（91%），绿色
  3. 待缴费：如 10 单，蓝色，显示待缴金额如"¥328,500"
  4. 逾期：如 4 单，红色，显示逾期金额如"¥89,200"
- 筛选区域：缴费状态下拉框（全部/未缴费/已缴费/逾期）、保单号搜索框、缴费截止日范围选择器、查询和重置按钮
- 主体：数据表格
- 底部分页器

数据表格列：
1. 保单号：如 SCH064257-262600，可点击查看详情
2. 客户名称：买方公司名称
3. 贸易类型：出口险（蓝色标签）/ 内贸险（紫色标签）
4. 保费金额：如 ¥15,200.00
5. 缴费截止日：如 2026-07-15
6. 实际缴费日：如 2026-07-10，未缴费显示"—"
7. 缴费状态：未缴费（蓝色标签）、已缴费（绿色标签）、逾期（红色标签带闪烁效果）
8. 剩余天数：
   - 未缴费且 >7 天：显示"还剩 X 天"（灰色）
   - 未缴费且 3-7 天：显示"还剩 X 天"（黄色）
   - 未缴费且 <3 天：显示"还剩 X 天"（橙色）
   - 已缴费：显示"—"
   - 逾期：显示"逾期 X 天"（红色加粗）
9. 操作列：
   - 未缴费：显示"发送提醒"按钮（蓝色）
   - 逾期：显示"紧急通知"按钮（红色）+ "标记已缴费"按钮（绿色）
   - 已缴费：显示"查看凭证"链接

紧急通知弹窗：
- 弹窗标题：发送紧急缴费通知
- 显示内容：保单号、客户名称、逾期天数、应缴金额
- 通知方式多选框：飞书消息（默认勾选）、邮件通知（默认勾选）、短信通知（默认不勾选）
- 通知对象：自动填充经办人和部门负责人，可手动添加
- 备注输入框
- 底部：取消按钮、发送通知按钮（红色）

逾期预警规则展示区域（页面底部）：
- 标题：预警规则配置
- 显示三条规则卡片：
  1. 黄色预警：缴费截止日前 7 天，通知经办人
  2. 橙色预警：缴费截止日前 3 天，通知经办人 + 部门负责人
  3. 红色预警：逾期 1 天，通知所有人（飞书 + 短信）
- 每条规则右侧有开关按钮，支持启用/禁用

技术要求：使用 React + Tailwind CSS + shadcn/ui 组件库，深色主题，响应式布局。
```

## 使用说明

1. 打开 [v0.dev](https://v0.dev)
2. 复制上面任意一个 Prompt
3. 粘贴到 v0.dev 的输入框
4. 点击生成，等待 UI 页面渲染
5. 如果效果不满意，可以继续对话微调，比如"把表格改成浅色主题"、"增加一个搜索框"等
6. 生成的代码可以直接导出为 React 组件

## 设计规范

- **主题**：深色主题（Dark Mode）
- **组件库**：shadcn/ui
- **样式**：Tailwind CSS
- **布局**：左侧菜单栏 + 右侧内容区（Dashboard 布局）
- **配色**：
  - 主色：蓝色 (#3B82F6)
  - 成功：绿色 (#22C55E)
  - 警告：黄色 (#EAB308)
  - 危险：红色 (#EF4444)
  - 信息：蓝色 (#3B82F6)
- **字体**：系统默认字体
- **间距**：使用 Tailwind 的标准间距（p-4, p-6, gap-4 等）
