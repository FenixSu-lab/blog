## 背景

事情是这样的，公司最近在搞会议室智能化升级，领导问我说飞书有个会议签到板的功能挺好用的，能不能给每个会议室都配上。

我一查，好家伙，飞书端的会议签到板，不算硬件，光软件费用一台设备就要 **999 元**。

![飞书会议签到板软件费用](https://fenixtc-1258871859.cos.ap-guangzhou.myqcloud.com/%E9%A3%9E%E4%B9%A6%E4%BC%9A%E8%AE%AE%E7%B3%BB%E7%BB%9F%E8%BD%AF%E4%BB%B6%E8%B4%B9%E7%94%A8.png)

公司有 **12 个会议室**，这么一算就是 **一万多**的软件费用，这还没算硬件呢。说实话这个价格确实有点离谱，就是一个日程展示 + 签到的功能，值这个价吗？

## 面临的问题

说白了需求其实很简单：

- 会议室门口放个平板，显示当前会议的主题、时间、参会人
- 支持签到功能，方便统计谁来了谁没来
- 能实时同步飞书日程，会议信息自动更新

功能不复杂，但飞书官方的报价确实让人肉疼。一万多对一个企业来说虽然不是小数目，但是买个软件授权就花这么多，多少有点不值当。

## 解决方案

既然飞书有开放 API，那我能不能自己搞一个？

调研了一下，飞书提供了 **CLI 工具**和完整的 **日程 API**，可以拉取会议室的日程信息。思路就很清晰了：

1. 通过飞书 API 对接飞书日程，拉取每个会议室的会议信息
2. 自研一个 Android 应用，部署到会议室门口的平板上
3. 应用实时展示当前/即将到来的会议信息，支持签到操作

说干就干，前后花了 **3 天时间**，从零开始把这套系统搞出来了。

## 开发细节

### 技术选型

- **开发工具**：Android Studio
- **语言 & UI 框架**：Kotlin + Jetpack Compose
- **JDK**：Java Embedded JDK 21
- **整体布局**：左侧会议室状态区 + 右侧固定日程栏

### 编译 & 部署

编译 Debug 包：

```powershell
$env:GRADLE_USER_HOME='C:\tmp\gradle-home'
.\gradlew.bat assembleDebug
```

平板开启开发者模式，USB 调试连接到 Android Studio，直接安装：

```powershell
$env:GRADLE_USER_HOME='C:\tmp\gradle-home'
.\gradlew.bat installDebug
```

### 调试排错

安装失败、闪退之类的问题，用 adb 日志一把梭：

```bash
adb devices
adb logcat -c
adb logcat | Select-String "AndroidRuntime|FATAL EXCEPTION|meetingroomdisplay"
```

基本上该报的错都在日志里了，Compose 的崩溃堆栈也比较清晰，定位问题不难。
