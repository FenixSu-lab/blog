## 写在前面

用了 Codex 一段时间了，从一开始的「这玩意能写代码？」到现在的「没有它我不会写代码了」，中间踩了不少坑，也摸索出一些玩法。

这篇文章不是官方文档的搬运，而是我自己实际使用中总结出来的 **实战经验**。有些功能官方文档讲不清楚，或者讲了但你不知道怎么用，这篇文章尽量把这些「隐性知识」写出来。

## 基础操作

### 计划模式

Codex 有一个「计划模式」，很多人不知道这个功能的存在。

普通模式下，你给一个需求，Codex 直接开写代码。但有时候需求比较复杂，你不想让它直接动手，而是先让它 **出一个方案给你看**。

这时候切到计划模式，Codex 会先分析需求、拆解任务、列出实现步骤，你确认没问题了再让它执行。相当于「先画图纸再施工」，避免它一上来就乱写一通，改都改不动。

### 观察 Agent 执行

Codex 执行任务的时候，你可以实时看到它在干什么——读了哪些文件、改了哪些代码、调用了什么工具。这个过程不是黑盒的，你可以随时介入。

如果发现它走偏了，直接打断，给它更明确的指令。别等它写完一大坨代码才发现方向错了，那成本就高了。

### 查看效果 + 迭代修改

写完之后怎么验证？几个常用操作：

- `/status` —— 查看当前额度信息和使用状态
- 设置里可以切换 **编程模式** 和 **普通模式**，编程模式更专注代码，普通模式更通用
- **计划模式** —— 前面说了，先出方案再执行
- **批注按钮** —— 可以直接在预览页面上批注修改，前端开发特别好用

## 常用功能

### 插件系统

Codex 有一个插件市场，里面有大量的编程类和工具类插件。

举几个例子：

- **Netlify 插件** —— 一键部署前端服务，不用手动配置 CI/CD
- **PPT 插件** —— 用 AI 生成演示文稿，做技术分享的时候特别省时间

插件的思路就是：你有什么重复性的工作，看看插件市场有没有现成的，没有的话自己写一个也不难。

### Browser Use

这个功能很强大——AI 可以操控浏览器。

具体能干什么呢？预览页面、自动点击、填写表单、滚动翻页、抓取数据。它通过 Chrome 扩展插件实现，相当于给 AI 装了一双「眼睛」和一双手。

实际使用场景：

- 自动化测试：让 AI 模拟用户操作，检查页面功能是否正常
- 数据抓取：给一个网址，自动提取需要的数据
- 表单填写：批量录入数据的时候，AI 比你快多了

### Computer Use

比 Browser Use 更进一步——AI 操控整个电脑。

微信、飞书、任何桌面应用，AI 都能操控。目前只支持 macOS 系统，这个限制暂时还没办法。

实际使用场景：

- 自动化办公操作：让 AI 帮你处理一些重复性的桌面操作
- 跨应用协作：比如从 Excel 里取数据，自动填到另一个系统里

> ⚠️ 注意：Computer Use 涉及桌面操控，权限管控要谨慎，生产环境慎用。

### Skills 技能包

Skills 是 Codex 的一个核心概念——把一组能力封装成一个「技能包」，需要的时候直接调用。

社区里有很多现成的技能：

- **Firecrawl** —— 联网搜索，让 AI 能访问互联网获取最新信息
- **Context7** —— 获取最新文档，写代码的时候能实时查阅官方文档
- **UI UX Pro Max** —— 美化前端页面，自动调整样式和布局
- **Remotion Best Practice** —— 动画制作的最佳实践

怎么安装？两种方式：

1. 用命令安装：`/Skill Installer remotion best practice`
2. 把 GitHub 链接丢给 Codex，它会自动识别并安装

**自定义技能** 也很简单。参考鱼皮的做法——他做了两件有意义的事情：

1. 把 AI 编程技巧写成了教程分享
2. 将平时使用的内容封装为 skills

封装一个 skill 需要定义清楚三件事：

- **这个技能要做什么** —— 明确能力边界
- **什么时候触发** —— 自动触发还是手动调用
- **需要注意的细节** —— 边界条件、异常处理

### MCP 连接外部服务

MCP（Model Context Protocol）是 AI 的「万能插头」——通过标准化协议连接外部工具和数据源。

比较实用的几个：

- **up-to-date Library** —— 获取最新的库文档和代码示例
- **防休眠功能** —— 运行时防止系统自动休眠，长时间执行任务的时候必备

MCP 的思路就是：AI 不是万能的，但通过 MCP 可以调用任何外部能力。需要什么能力，接一个 MCP server 就行。

## 进阶功能

### 上下文和对话管理

上下文管理分三个层级：

**全局配置** —— 所有对话都生效的设置。比如在个性化设置里写上「中英文切换、尽量减少输出内容」，每次对话都会带上这些偏好。

**项目专属配置** —— 在项目根目录建立 `AGENTS.md` 文件，定义这个项目的专属约定。比如技术栈、编码规范、目录结构等，Codex 每次在这个项目里工作都会自动读取。

**自动记忆** —— Codex 会自动记住你的使用习惯和偏好，越用越顺手。

### 桌面宠物

是的，你没看错，Codex 有桌面宠物功能。

用 `/Hatch Pet` 命令可以自定义创建宠物，也可以安装其他人分享的宠物。纯粹是趣味功能，但偶尔看看还挺治愈的。

### 持久记忆系统

和自动记忆不同，持久记忆是你可以主动管理的知识库。把项目的关键信息、设计决策、踩过的坑记下来，Codex 在后续开发中会参考这些记忆。

### 代码审查和版本管理

这里有一个很实用的功能——**工作树模式（Worktree）**。

允许多个 Agent 在同一个项目上并行工作，每个 Agent 在自己的分支里互不干扰。相当于 AI 版本的「多人协作」，一个人负责前端、一个人负责后端、一个人负责测试，同时推进。

### 定时自动化

支持设置定时任务模板，让 AI 在指定时间自动执行重复性工作。比如每天早上自动拉取最新的依赖、每周自动跑一次代码质量检查。

### 手机远程操作

通过手机远程操控 Codex，出门在外也能让 AI 帮你干活。虽然大部分场景用不太上，但偶尔应急的时候很方便。

## 高阶玩法

### SubAgents 并行加速

一个复杂任务拆成多个子任务，分配给多个 SubAgent 同时执行。比如一个全栈项目，前端、后端、数据库三个 Agent 同时开工，效率直接翻倍。

### Fork 分叉的妙用

对当前的对话或任务进行分叉，保留一个版本的同时尝试不同的方向。相当于 Git 的分支概念用在了 AI 对话上——「这个方案试试看，不行我再切回去」。

### 自定义模型

Codex 支持接入自定义模型。如果你有自己的模型服务，或者想用特定领域微调过的模型，可以通过配置接入。

### Hooks 生命周期钩子

在 Agent 执行的不同阶段插入自定义逻辑。比如代码生成后自动跑 lint、文件保存后自动触发构建、任务完成后自动发通知。

*AI 编程工具更新很快，但核心思路不会变——让 AI 处理重复性工作，人专注于创造性思考。Codex 只是工具，怎么用好它，取决于你对业务的理解深度。*
